## Цели проекта  
Дипломная работа поможет закрепить знания и навыки по автоматизации тестирования мобильных приложений.  

## В результате выполнения решены следующие задачи:  
1. проведено ручное тестирование мобильного приложения «Мобильный хоспис».
2. составлен чек-лист для проверки приложения.
3. составлены тест-кейсы для проверки приложения.
4. проведена автоматизация проверки тест-кейсов по чек-листу.
5. составлены отчёты о тестировании.

### *	Позитивные сценарии:
1.	Покупка тура за собственные средства (вводим валидные данные)
2.	Покупка тура в кредит (вводим валидные данные)
3.	Пользователь вводит валидные данные, выбирает оплату по карте, на карте при этом недостаточно средств для выполнения операции - происходит отказ в операции(по условиям для этого и следующего сценария дан номер карты по которому операция будет отклонена);
4.	Пользователь вводит валидные данные, выбирая оплату в кредит - происходит отказ в выполнении операции.
### *	Негативные сценарии:
1.	Пользователь выбирает покупку с оплатой по карте, заполняет невалидными данными поля;
2.	Пользователь выбирает покупку с оплатой в кредит, заполняет невалидными данными поля;
•	Из перечисленных негативных сценариев проверяем, что:
o	появляются подсказки о том, что поле неверно заполнено;
o	приложение не передает форму с невалидными данными в платежный сервис, платеж не проходит;
o	отдельно проверим каждое поле, заполнив его данными в неверном формате.
o	2 варианта неполного номера карты - 15 знаков и 1 знак;
o	неверный формат месяца;
o	неверный формат года;
o	неверный cvc код,
o	ввод цифр в поле имя пользователя;
o	ввод пустого поля имя пользователя.

## Перечень используемых инструментов и обоснование
1. *JDK 11* - комплект для разработки программного обеспечения) на Java — набор инструментов для разработки программного обеспечения, которые содержит комплект необходимых библиотек, компилятор, отладчик, необходимые для разработку автоматизированных тестов;  
2. *Android Studio* - инструмент позволяющией создавать автоматизированные тесты, разворачивать эмуляторы мобильных устройств, выявлять идентификаторы элементов приложения;  
3. *IntelliJ IDEA* - среда разработки, ориентированая на Java-кодинг;
4. *Espresso* - библиотека, интегрированная с Android Studio для создания автотестов мобильных приложений;
5. *Appium* - свободно распространяемый фреймворк с открытым исходным кодом, предназначенный для тестированиz пользовательского интерфейса мобильных приложений для iOS и Android;  
6. *Facker* - библиотека для генерации вводных данных;  
7. *Cypress* - позволяет не только создавать автотесты, но и может быть использована для поиска идентификаторов элементов приложения;  
8. *Система контроля версий* - скорее всего Git и Github - необходимо для хранения автотестов, обеспечивают возможности откатов к предыдущим версиям, позволяют реализовывать CI;
9. *Allure* - инструмент автоматизации предоставления отчетов в графической форме. 

Это ключевые инструменты тестирования. Вполне возможно использовать для иных целей такие как MonkeyRunner, Espresso, снифферы, фаззи-тестирование и прочее.

## Перечень и описание возможных рисков при тестирования
* несоответствие системных устройств, на которых предполагается провести тестирования, требованиям приложения;
* ложные срабатывания тестов: выявление багов, когда их нет и наоборот;    
* определена неверная логика работы приложения;    
* могут быть протестированы не все сценарии;
* высокие временные затраты;
* зависимость автотестов от устройств (не все объекты могут быть видимы, например);
* использование неудачных тестовых данных или недостаточного количества тестовых данных;
* недостаточный для решения поставленной задачи профессиональный уровень тестировщика.

### Интервальная оценка реализации тестирования с учётом рисков (в часах)
*	подготовка окружения, установка приложения на эмулятор и мобильное устройство:  
  1. 2 часа (при отсутствии дополнительных рисков);
  2. 4 час (при наличии рисков);
* исследование приложения и подготовка тестовой документации: чек-листа, тест-кейсов, сценариев:  
  1. 40 часов (при отсутствии дополнительных рисков);
  2. 60 часов (при наличии рисков);
*	написание автотестов, тестирование и отладка автотестов:  
  1. 24 часа (при отсутствии дополнительных рисков);
  2. 40 часов (при наличии рисков);
*	формирование и анализ отчетов - 8 часов.  
Таким образом временные затраты на реализацию проекта могут колебаться от 74 рабочих часов до 112 рабочих часов.
